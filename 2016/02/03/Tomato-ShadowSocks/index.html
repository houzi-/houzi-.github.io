Open JFFS or mount U disk

JFFS

Administration >> JFFS >> Format/Erase >> save

U disk
note:If you do not support JFFS you can choose to mount U disk installed

Enter fdisk -l /dev/sda to see U disk drive which mounts:

root@unknown:/#
Disk /dev/sda: 15.9 GB, 15931539456 bytes
255 heads, 63 sectors/track, 1936 cylinders
Units = cylinders of 16065 * 512 = 8225280 bytes

Device Boot Start End Blocks Id System
/dev/sda1 1 1937 15558112+ c Win95 FAT32 (LBA)

I tagged by the above information indicates that the U disk drive is mounted /dev/sda1

Enter umount -l /dev/sda1 unloading U disk mounted partition , so you can easily format

Enter mkfs.ext3 /dev/sda1 to format the partition as ext3/4 file system format

Modify shadowsocks script installation path

note: After downloading unzip into shadowsocks.zip router /tmp directory

Looking script this command:PREFIX=/jffs

  1. If it is mounted JFFS, you should keep the default
  2. If it is mounted U disk, You should change the PREFIX=/mnt/sda1

Installation script shadowsocks

Download:shadowsocks-libev_2.4.3-1-RC、chinadns_1.3.2、pdnsd_1.2.9a、pcap-dnsproxy0.4.5.0-1、resolvip_0.1、diffutils3.3-3upload to the router /tmp directory

Enter chmod +x /tmp/shadowsocks given execute permissions
Installation /tmp/shadowsocks set_up

Configuration shadowsocks Account

note:Local Server Port and Sever Port Do not backwards

Select agent program

Option
mipsel+armv7
1
ss-redir+gfwlist+pdnsd

3
ss-redir+chnroutes+pcap_dnsproxy
4
ss-tunnel+chnroutes+chinadns

I’m currently using Option 3 , Option 3 is recommended
note:

  1. If the configuration is successful you can use directly shadowsocks Agent
  2. If you use Option 3 appears unable to use shadowsocks agents may be Pcap_DNSProxy process dies , so you should enter /opt/etc/init.d/S27pcap_dnsprox start

Reboot Router
